## 2. Правила граматики

Граматика описує конструкцію `if-else`:

| №   | Правило                              |
|-----|--------------------------------------|
| 1   | `I -> If ( A ) { B ; R } C`          |
| 2   | `A -> exp`                           |
| 3   | `B -> smth`                          |
| 4.1 | `R -> B ; R`                         |
| 4.2 | `R -> $`                             |
| 5.1 | `C -> Else X`                        |
| 5.2 | `C -> $`                             |
| 6.1 | `X -> { B ; R }`                     |
| 6.2 | `X -> I`                             |

**Нетермінали:** `I, A, B, R, C, X`  
**Термінали:** `If, (, ), {, }, ;, exp, smth, Else`  
**Епсилон:** `$`  
**Маркер кінця:** `#`

## 4. Конфігурація магазинного автомата

Магазинний автомат визначається як: **M = {P, S, s₀, f, F, H, h₀}**, де:

- **P** — вхідний алфавіт: `{If, (, ), {, }, ;, exp, smth, Else, #}`
- **S** — алфавіт станів: `{s₀, s}`
- **s₀** — початковий стан
- **F** — множина кінцевих станів: `{s}`
- **H** — алфавіт магазинних символів: `{I, A, B, R, C, X, If, (, ), {, }, ;, exp, smth, Else, h₀}`
- **h₀** — маркер дна магазину
- **f** — функція переходів (команди автомата)

**Початкова конфігурація:**

```
(s₀, µ, h₀I)
```

де `µ` — вхідний ланцюжок, `h₀` — маркер дна магазину, `I` — стартовий символ граматики.

**Заключна конфігурація:**

```
(s, $, $)
```

Функція переходів буває двох типів:
- **f** — зі зрушенням вхідної головки (термінал з вхідної стрічки споживається)
- **f\*** — без зрушення вхідної головки (лексема залишається у вхідній стрічці)

## 5. Побудова команд магазинного автомата

Команди будуються на основі правил граматики та множини ВИБІР за чотирма типами.

### Тип 1 — правило починається з термінала `A -> b α`

> f(s₀, b, A) = (s, дзеркало(α))

**Правило 1:** `I -> If ( A ) { B ; R } C`  
Перший символ — `If` (термінал). Решта у зворотному порядку: `C } R ; B { ) A (`.  
`f(s₀, If, I) = (s, C } R ; B { ) A ()`

**Правило 2:** `A -> exp`  
`f(s₀, exp, A) = (s, $)`

**Правило 3:** `B -> smth`  
`f(s₀, smth, B) = (s, $)`

**Правило 5.1:** `C -> Else X`  
Перший символ — `Else` (термінал). Решта у зворотному порядку: `X`.  
`f(s₀, Else, C) = (s, X)`

**Правило 6.1:** `X -> { B ; R }`  
Перший символ — `{` (термінал). Решта у зворотному порядку: `} R ; B`.  
`f(s₀, {, X) = (s, } R ; B)`

### Тип 2 — правило починається з нетермінала `A -> B α`

> f\*(s₀, x, A) = (s, дзеркало(B α)),  x ∈ ВИБІР(A -> B α)

**Правило 4.1:** `R -> B ; R`, ВИБІР = `{smth}`  
`f*(s₀, smth, R) = (s, R ; B)`

**Правило 6.2:** `X -> I`, ВИБІР = `{If}`  
`f*(s₀, If, X) = (s, I)`

### Тип 3 — анулююче правило `A -> $`

> f\*(s₀, x, A) = (s, $),  x ∈ ВИБІР(A -> $)

**Правило 4.2:** `R -> $`, ВИБІР = `{}}`  
`f*(s₀, }, R) = (s, $)`

**Правило 5.2:** `C -> $`, ВИБІР = `{#}`  
`f*(s₀, #, C) = (s, $)`

### Тип 4 — команди для терміналів у середині або в кінці правил

> f(s₀, b, b) = (s, $)

Для кожного термінала, що зустрічається в правих частинах правил:

```
f(s₀, (, ()    = (s, $)
f(s₀, ), ))    = (s, $)
f(s₀, ;, ;)    = (s, $)
f(s₀, Else, Else) = (s, $)
f(s₀, If, If)  = (s, $)
f(s₀, exp, exp) = (s, $)
f(s₀, smth, smth) = (s, $)
f(s₀, {, {)   = (s, $)
f(s₀, }, })   = (s, $)
```

### Тип 5 — перехід у заключний стан

`f*(s₀, $, h₀) = (s, $)`

## 7. Перевірка роботи магазинного автомата

### 7.1 Ланцюжок 1 — простий `if` без `else`

**Вхідний рядок:**
```
if ( expr ){
    stmt1;
}
```

**Граматичні символи після сканування:**
```
If ( exp ) { smth ; }
```

**Зміна конфігурацій:**

```
(If(exp){smth;}#, Ih₀)              ├ 1
((exp){smth;}#, (A){B;R}Ch₀)        ├ 10
(exp){smth;}#, A){B;R}Ch₀)          ├ 2
(){smth;}#, ){B;R}Ch₀)              ├ 11
({smth;}#, {B;R}Ch₀)                ├ 17
(smth;}#, B;R}Ch₀)                  ├ 3
(;}#, ;R}Ch₀)                       ├ 12
(}#, R}Ch₀)                         ├ 5
(}#, }Ch₀)                          ├ 18
(#, Ch₀)                            ├ 7
(#, h₀)                             ├ 19
($, $)
```

**Результат:** ✅ ЛАНЦЮЖОК РОЗПІЗНАНО

---

### 7.2 Ланцюжок 2 — `if` з `else` та кількома операторами

**Вхідний рядок:**
```
if ( expr ){
    stmt;
}
else {
    stmt;
    stmt;
    stmt;
}
```

**Граматичні символи після сканування:**
```
If ( exp ) { smth ; } Else { smth ; smth ; smth ; }
```

**Зміна конфігурацій:**

```
(If(exp){smth;}Else{smth;smth;smth;}#, Ih₀)                ├ 1
((exp){smth;}Else{smth;smth;smth;}#, (A){B;R}Ch₀)           ├ 10
(exp){smth;}Else{smth;smth;smth;}#, A){B;R}Ch₀)             ├ 2
(){smth;}Else{smth;smth;smth;}#, ){B;R}Ch₀)                 ├ 11
({smth;}Else{smth;smth;smth;}#, {B;R}Ch₀)                   ├ 17
(smth;}Else{smth;smth;smth;}#, B;R}Ch₀)                     ├ 3
(;}Else{smth;smth;smth;}#, ;R}Ch₀)                          ├ 12
(}Else{smth;smth;smth;}#, R}Ch₀)                            ├ 5
(}Else{smth;smth;smth;}#, }Ch₀)                             ├ 18
(Else{smth;smth;smth;}#, Ch₀)                               ├ 6
({smth;smth;smth;}#, Xh₀)                                   ├ 8
(smth;smth;smth;}#, B;R}h₀)                                 ├ 3
(;smth;smth;}#, ;R}h₀)                                      ├ 12
(smth;smth;}#, R}h₀)                                        ├ 4
(smth;smth;}#, B;R}h₀)                                      ├ 3
(;smth;}#, ;R}h₀)                                           ├ 12
(smth;}#, R}h₀)                                             ├ 4
(smth;}#, B;R}h₀)                                           ├ 3
(;}#, ;R}h₀)                                                ├ 12
(}#, R}h₀)                                                  ├ 5
(}#, }h₀)                                                   ├ 18
(#, h₀)                                                     ├ 19
($, $)
```

**Результат:** ✅ ЛАНЦЮЖОК РОЗПІЗНАНО

---

### 7.3 Ланцюжок 3 — вкладені `if-else if-else`

**Вхідний рядок:**
```
if ( expr ){
    stmt;
    stmt;
}
else if ( expr ){
    stmt;
    stmt;
}
else if ( expr ){
    stmt;
}
else {
    stmt;
    stmt;
    stmt;
}
```

**Граматичні символи після сканування:**
```
If ( exp ) { smth ; smth ; } Else If ( exp ) { smth ; smth ; }
Else If ( exp ) { smth ; } Else { smth ; smth ; smth ; }
```

**Зміна конфігурацій (фрагмент з ключовими кроками):**

```
(If(exp){smth;smth;}ElseIf(exp){smth;smth;}ElseIf(exp){smth;}Else{smth;smth;smth;}#, Ih₀) ├ 1
((exp){smth;smth;}...#, (A){B;R}Ch₀)    ├ 10
...
(ElseIf(exp){smth;smth;}...#, Ch₀)      ├ 6   ← C -> Else X
(If(exp){smth;smth;}...#, Xh₀)          ├ 9   ← X -> I (без зрушення)
(If(exp){smth;smth;}...#, Ih₀)          ├ 1   ← рекурсивно розбираємо if
...
(ElseIf(exp){smth;}...#, Ch₀)           ├ 6
(If(exp){smth;}...#, Xh₀)               ├ 9
(If(exp){smth;}...#, Ih₀)               ├ 1
...
(Else{smth;smth;smth;}#, Ch₀)           ├ 6
({smth;smth;smth;}#, Xh₀)              ├ 8   ← X -> { B ; R }
...
(#, h₀)                                 ├ 19
($, $)
```

**Результат:** ✅ ЛАНЦЮЖОК РОЗПІЗНАНО

## 9. Опис програмної реалізації

Програма реалізована мовою **Python** і складається з таких модулів:

**`scanner(source)`** — лексичний аналізатор на основі скінченного автомата. Розбиває вхідний рядок на токени типів `KEYWORD`, `IDENTIFIER`, `DELIMITER`.

**`tokens_to_grammar_symbols(tokens)`** — перетворює токени у граматичні символи: `if` → `If`, `else` → `Else`, ідентифікатори типу `expr*` → `exp`, інші ідентифікатори → `smth`.

**`build_pda_commands(choice)`** — будує команди МА за чотирма типами відповідно до методичних вказівок. Повертає список словників з полями `type`, `input`, `stack`, `result`.

**`simulate_pda(input_tokens, commands, label)`** — симулює роботу магазинного автомата: на кожному кроці знаходить відповідну команду за поточною лексемою та вершиною магазину, виконує перехід і виводить конфігурацію. Фіксує результат — розпізнано чи помилка.
